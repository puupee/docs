# 小汪盘点子应用设计

日期：2026-05-12

## 背景

创建新的 Puupee 子应用“小汪盘点”，英文应用 ID 为 `inventory`，包名为 `puupee_inventory`，路由根路径为 `/inventory`。应用定位为个人资产中枢，用于盘点生活中的实物资产和虚拟资产，并通过 Puupee/Sync 持久化和同步。

第一版采用“资产中枢 MVP”路线：实物和虚拟资产使用统一模型；拍照和截图入口生成识别草稿，用户确认后入库；真实 OCR/AI 暂不接入，先用可替换的模拟识别服务完成端到端流程。

## 目标

- 新增一个可运行的 Flutter 子应用 `apps/puupee/inventory`。
- 新增 Sync 原生功能模型 `InventoryAsset`，contentType 为 `application/vnd.puupee.inventory.asset`。
- 支持个人实物资产和虚拟资产的统一录入、展示、筛选、详情编辑、提醒和轻量盘点。
- 支持三种添加入口：手动添加、拍照添加实物、截图添加虚拟资产。
- 拍照和截图第一版走模拟识别服务，生成可编辑草稿，确认后保存到 Puupee/Sync。
- 复用仓库已有启动、路由、设置、同步和 shadcn Flutter UI 约定。

## 非目标

- 不接真实 OCR、视觉模型或外部 AI 服务。
- 不做企业资产审计、审批、盘点批次、责任人流转。
- 不做多人家庭共享和借用人管理。
- 不做复杂折旧、保险理赔、维修工单或资产处置流程。
- 不直接复用 `device`、`billing.period` 等模型作为主数据源；这些可在后续作为关联或导入来源。

│   ├── env.dart

│   ├── components/

应用使用 `runMyApp()` 启动，`InventoryEnvConfig` 配置 `appId: inventory` 和 `appTitle: 小汪盘点`。路由根路径为 `/inventory`，桌面端使用侧栏布局，移动端使用底部导航。

### Sync 功能模型

在 `packages/puupee/lib/src/features/` 增加 `inventory_asset.dart`，使用 `@PuupeeFeature(contentTypeOverride: 'application/vnd.puupee.inventory.asset')` 声明模型，并通过代码生成得到 `InventoryAsset` 类型、contentType 常量、CLI/MCP schema。

`InventoryAsset` 统一承载实物和虚拟资产。核心字段分组如下：

- 身份字段：名称、描述、资产类型、分类、状态、标签、重要程度。
- 实物属性：品牌、型号、序列号、位置、保管方式、附件关系。
- 虚拟属性：服务名、账号线索、套餐、授权类型、访问入口。
- 价值字段：购买价、当前估值、币种。
- 生命周期字段：购买日期、保修到期、续费/到期日期、订阅周期。
- 来源字段：手动、拍照、截图、导入；可记录来源文件或截图线索。
- 提醒字段：提醒开关、提醒日期、提醒类型。

字段会映射到 Puupee 通用 slot（`s1`-`s11`、`i1`-`i11`、`r1`-`r11`、`d1`-`d11`、`b1`-`b11`）。实现时优先保证第一版所需字段，超出 slot 的结构化信息可放在 `content` JSON 中，避免模型一开始过度膨胀。

### 数据流

```text
手动填写 / 拍照 / 截图
  -> InventoryRecognitionService
  -> InventoryAssetDraft
  -> 用户确认编辑
  -> InventoryAssetRepo
  -> Puupee/Sync 持久化与同步
  -> 总览、列表、提醒、盘点视图
```

`InventoryRecognitionService` 是可替换接口。第一版实现 `MockInventoryRecognitionService`，根据入口类型返回稳定的示例草稿。后续接入真实 OCR/AI 时替换服务实现，不改确认页和保存逻辑。

## 资产类型

第一版内置以下个人资产类型：

- 实物：手机、电脑、平板、耳机、充电器、充电线、家电、家具、工具、书籍、票据、收藏品、耗材。
- 虚拟：App 会员、订阅服务、软件授权、云服务、域名、服务器、礼品卡、余额、积分权益、电子票券、保修凭证、证件资料、账号资料。

实物资产重点关注位置、状态、保修、附件、价值和盘点复核。虚拟资产重点关注账号线索、套餐、金额、周期、到期/续费日和提醒。

## 录入流程

### 手动添加

用户选择实物或虚拟类型后进入表单。表单按类型显示不同字段，但最终保存为同一个 `InventoryAsset`。

### 拍照添加实物

用户从拍照入口进入后，第一版展示模拟拍照/选择图片流程，调用模拟服务生成一个或多个实物草稿。例如一张照片可生成“手机”“充电器”“充电线”等候选项。确认页支持改名、选择分类、补位置、补价格、补保修、删除误识别项。

### 截图添加虚拟资产

用户从截图入口进入后，第一版展示模拟截图导入流程，调用模拟服务生成虚拟资产草稿。例如从会员页或订单页识别服务名、套餐、金额、周期、下次续费日、账号线索。续费类资产默认开启提醒。

### 确认页

所有识别结果必须先进入确认页，不直接入库。确认页展示来源、识别字段、低置信提示、缺失字段和保存按钮。字段来源分为识别、用户填写、缺失，便于后续接真实识别服务时保留透明度。

## 界面

### 导航

第一版主导航：

- 总览
- 资产
- 提醒
- 盘点
- 我的

桌面端使用侧栏和主内容区；移动端使用底部导航。设置相关页面复用共享设置栈。

### 总览页

展示资产总数、实物数量、虚拟数量、本月续费、即将过保、待确认草稿等统计。提供拍照实物、导入截图、手动添加三个快捷入口。

### 资产列表页

统一展示实物和虚拟资产，支持搜索、类型筛选、分类筛选、状态筛选、位置筛选和标签筛选。列表项展示名称、类型、状态、位置或账号线索、下一提醒日期。

### 资产详情/编辑页

展示并编辑基础信息、资产属性、价值、生命周期、提醒、来源和备注。状态支持在用、闲置、遗失、已处理、已过期。

### 提醒页

聚合即将续费、即将到期、即将过保、需要复核的资产。第一版只做列表和状态更新，不做系统级通知调度。

### 盘点页

轻量盘点视图，按位置、分类、状态列出需要复核的资产。用户可快速标记“已确认”“需处理”“找不到”。不做企业级盘点批次。

## 错误处理

- 识别失败时显示可编辑空草稿，允许用户继续手动填写。
- 保存失败时保留草稿内容并显示统一消息提示。
- Sync 未就绪时使用现有同步层的错误状态展示，页面不直接访问 drift。
- 低置信字段不阻止保存，但在确认页提示用户复核。
