## 文档维护原则

用户文档应回答“这是什么、我能用它做什么、怎样开始、遇到问题怎么办”。开发文档应记录“为什么这样实现、迁移过程、验证记录、内部任务和风险”。

## 放置新文档

| 新文档类型 | 放置位置 |
| --- | --- |
| 面向用户的功能介绍 | `docs/` |
| 面向用户的操作指南 | `docs/` |
| SDK 安装和公开 API 用法 | `docs/` 或生成客户端的 `doc/` |
| 内部实现方案 | `dev/` |
| 迁移计划和实现总结 | `dev/` |
| 测试计划和排障记录 | `dev/` |

## 本地预览

在根文档站目录运行 Mintlify：

```bash
cd docs
mint dev
```

预览站点只配置用户可见页面。`dev/` 目录中的内容不进入导航。
