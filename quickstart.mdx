## 选择入口

<Steps>
  <Step title="选择你要使用的能力">
    如果你是终端用户，从应用文档开始；如果你要集成 Puupee 能力，从包与 SDK 或公开接口开始。
  </Step>
  <Step title="查看对应目录下的 docs">
    每个子应用和子包都保留自己的 `docs/` 目录。这里放置功能说明、价格说明、公开 API 说明和使用指南。
  </Step>
  <Step title="避开 dev 目录">
    `dev/` 目录包含实现细节、迁移记录和内部任务，不属于用户文档，也不会出现在 Mintlify 导航中。
  </Step>
</Steps>

## 常用文档位置

| 你想找 | 位置 |
| --- | --- |
| 应用功能与价格 | `apps/puupee/<app>/docs/FEATURES.md`、`apps/puupee/<app>/docs/PRICING.md` |
| Dart/Flutter 包说明 | `packages/<package>/README.md` 或 `packages/<package>/docs/` |
| TypeScript/Go API 客户端 | `packages/puupee-api-axios/docs/`、`packages/puupee-api-go/docs/` |
| Dart API 客户端 | `packages/puupee_api_client/doc/` |
| 同步节点接口 | `packages/puupee_sync_api/doc/` |

<Note>
  维护文档时，请先判断受众。用户能直接理解并使用的内容放入 `docs/`；只服务开发、排障、重构或历史追踪的内容放入 `dev/`。
</Note>
